### 1.2 API 请求与响应规范

#### 响应格式

```json
// 成功响应
{
    "code": 0,
    "message": "success",
    "data": { ... }
}

// 分页列表响应
{
    "code": 0,
    "message": "success",
    "data": {
        "items": [ ... ],
        "total": 100,
        "page": 1,
        "page_size": 20
    }
}

// 错误响应
{
    "code": 1,
    "message": "错误描述信息",
    "data": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 0=成功，1=业务错误，-1=未登录，-2=无权限 |
| `message` | string | 提示信息 |
| `data` | object/array/null | 返回数据 |

## 二、代码风格要求

### 2.1 Python 后端规范

#### 命名规范

| 类别 | 规范 | 示例 |
|------|------|------|
| 文件名 | `snake_case` | `admin.py`, `user_repository.py` |
| 类名 | `PascalCase` | `UserRepository`, `LoginHandler` |
| 函数/方法 | `snake_case` | `get_user_by_username()` |
| 变量 | `snake_case` | `password_hash`, `url_template` |
| 常量 | `UPPER_SNAKE_CASE` | `DEFAULT_PAGE_SIZE = 20` |

#### 缩进

- 统一使用 **制表符（Tab）** 缩进
- 行尾不留空格

### 2.2 前端规范

#### 模板文件

- 文件名：`snake_case`（如 `admin_user_list.html`）
- 继承 `base.html` 或 `admin.html`
- HTML 缩进使用 **2 空格**
- CSS class 使用 `-` 连接

```html
{% extends "admin.html" %}

{% block content %}
<div class="xxx-container">
    <h2>模块标题</h2>
</div>
{% end %}
```

### 2.3 数据库规范

| 维度 | 规范 |
|------|------|
| 表名 | `snake_case`，复数形式（如 `users`, `lookout_records`） |
| 主键 | `id`，INTEGER 自增 |
| 时间字段 | `create_at`, `update_at`，DATETIME 类型 |
| 状态字段 | `status`，INTEGER（1=启用，0=禁用） |
| 外键 | `xxx_id` 命名，如 `role_id`, `source_id` |

## 三、模块对接流程

### 3.1 模块依赖顺序

```
第一梯队（基础模块）:
  认证模块 ← 用户管理 ← 功能/角色/权限管理 ← 模型引擎

第二梯队（业务模块）:
  瞭望管理 → 深度采集
  数字员工 → 技能管理
  模型引擎 → 对话功能 → 会话/对话管理

第三梯队（用户侧）:
  对话功能 + 问数功能 + @数字员工 + PDF导出 + 注册

第四梯队（高级功能）:
  数智大屏 + 智能舆情 + 多数据库支持
```

### 3.2 对接步骤

```
Step 1: 提供方完成接口开发并自测
    ↓
Step 2: 提供方将代码合并到 develop 分支
    ↓
Step 3: 调用方拉取最新代码（git pull）
    ↓
Step 4: 调用方编写调用代码并测试
    ↓
Step 5: 双方联调，验证数据流
    ↓
Step 6: 联调通过，关闭对接任务
```
