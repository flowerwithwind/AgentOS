## 六、任务进度跟踪机制

### 6.1 每日站会

- **时间**：每天 10:00
- **议程**：昨日完成 → 今日计划 → 阻塞问题 → 需要协助
- **记录**：组长更新进度看板

### 6.2 进度状态

| 状态 | 说明 |
|------|------|
| 🔴 待开始 | 任务未启动 |
| 🟡 进行中 | 任务正在开发 |
| 🔵 待审核 | 开发完成，等待代码审查 |
| 🟢 已完成 | 审查通过，合并到主分支 |

### 6.3 代码审查流程

1. 开发完成 → 提交 Pull Request 到 `develop`
2. 组长 4 小时内审查
3. 通过 → 合并；不通过 → 返回修改
